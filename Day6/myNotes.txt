exercise 4.3:
global alignment compares entire sequences end-to-end, so mismatching flanks and length differences reduce the total score
local alignment focuses on the best conserved fragment only, so it usually gives a higher score when only part of sequences is homologous
biological interpretation: these proteins likely share a conserved local motif/domain, but are not uniformly similar across full length

exercise 5:
heuristic aligners (BLAST, minimap2) use seed-and-extend instead of evaluating all DP cells like full Smith-Waterman
computational gain:
- much faster runtime
- much lower memory usage
- scalable to very large databases and long-read datasets
trade-off / what is lost:
- no guarantee of globally optimal local alignment
- reduced sensitivity for weak/divergent homology (especially if seeds are sparse or disrupted by many mismatches/indels)
- results depend on heuristic parameters (k-mer size, seed density, chaining thresholds), so accuracy-speed balance must be tuned

summary:
heuristics buy speed and scalability, but sacrifice optimality guarantees and some sensitivity

exercise 6.3:
N in RNA-seq splice-aware alignment means a skipped region on the reference (typically an intron between exons)
it represents transcript splicing: the read is contiguous in RNA/cDNA, but alignment jumps over intronic genomic sequence
D (deletion) means a true gap relative to the reference within one local alignment block, usually a small indel event
key difference: N models exon-exon junction / long genomic skip, D models sequence-level deletion in the aligned region

exercise 6.4:
a standard contiguous DNA aligner like bwa mem assumes mostly colinear alignment with short indels and does not explicitly model splice junctions
for a read spanning an exon-exon junction with a long intron-sized gap (1500N), such aligner may:
- fail to align fully
- split/soft-clip the read
- force suboptimal gapped alignment
therefore RNA-seq reads spanning introns require splice-aware aligners (e.g. STAR, HISAT2, minimap2 -x splice)

exercise 7.1:
standard reference-based RNA-seq quantification (e.g. STAR + featureCounts) performs full base-level alignment to genomic coordinates, then assigns aligned reads/fragments to genes/features
pseudo-alignment (kallisto/salmon) skips full DP/base-level alignment and instead maps reads to compatible transcripts using k-mer/transcriptome indexing and probabilistic assignment
paradigm shift:
- from exact read-to-genome alignment + counting
- to lightweight compatibility mapping + statistical abundance inference
practical effect: much faster and lower-memory quantification, especially for large cohorts

exercise 7.2:
main limitation of pseudo-alignment: loss of explicit base-resolution genomic positioning (no full CIGAR-level alignment to the genome)
you lose or strongly limit:
- precise genomic coordinates of each read
- direct inspection of splice junction placement per read
- robust variant/indel calling from read alignments
- structural-event analyses requiring exact mapping context (novel splice sites, fusions, allele-specific mapping details)
in short: excellent for transcript quantification, weaker when downstream tasks need exact per-base genomic alignments

exercise 8.2.2:
HISAT2 must use genomic DNA FASTA (reference genome), because it aligns reads to genomic coordinates and splice-aware exon-intron structure
Kallisto must use transcript cDNA FASTA (transcriptome), because pseudo-alignment is performed against transcript sequences

exercise 8.2.3:
HISAT2 benefits from GTF-derived splice sites because known exon junctions improve splice-aware genomic alignment sensitivity and accuracy
Kallisto ignores GTF at indexing because its index is built directly from transcript sequences; exon structure is already implicit in transcript FASTA entries
in other words, transcriptome FASTA already encodes the spliced products, so a separate exon-intron annotation is not required for index construction

exercise 8.4:
pipeline required to discover a novel unannotated splice junction: HISAT2 (or another splice-aware genome aligner)
why:
- novel junction discovery requires read-level genomic split alignments crossing exon-exon boundaries not present in GTF
- HISAT2 can align to the genome with split CIGARs (N) and reveal previously unannotated junction positions
why Kallisto fails for this specific task:
- pseudo-alignment is performed against known transcript sequences in the transcriptome index
- if junction/transcript is absent from annotation/transcript FASTA, it cannot be explicitly discovered as a new genomic junction event
