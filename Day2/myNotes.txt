cat results/gc_compare.txt

The weighted mean from the Python script does not match the average calculated from the EMBOSS output.
The Python script computes GC content as a weighted mean across all nucleotides in the dataset,
meaning each nucleotide contributes equally to the final value. In contrast, EMBOSS reports GC content per sequence.
When those per-sequence GC values are averaged, each sequence contributes equally regardless of its length.
Since transcript lengths vary significantly, the arithmetic mean of per-sequence GC differs from the global weighted GC.
The difference therefore results from weighted versus unweighted averaging.

7.4

Scaling
The memory usage of the Python script remains relatively stable across different input sizes.
Peak memory does not increase proportionally with file size,
indicating that the implementation processes sequences in a streaming manner rather than storing the entire dataset in memory.
Therefore, memory usage does not scale linearly with total file size or with the length of the longest sequence.
Time, however, increases roughly proportionally to input size.

Implementation
The /usr/bin/time binary is required because it reports detailed resource usage,
including maximum resident set size (peak memory, %M).
The built-in shell time command typically reports only execution time and does not provide reliable memory statistics.
Therefore, /usr/bin/time is necessary for accurate memory benchmarking.
